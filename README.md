# CLI1 - כלי אריזת קבצי קוד 📦

כלי שורת-פקודה (CLI) ליצירת bundle של קבצי קוד מתיקייה מסוימת לקובץ אחד מאוחד.

## תיאור

**CLI1** הוא כלי שימושי לפיתוחים בעברית המאפשר לך:
- **אריזת קבצים** - שילוב קבצי קוד מתכנות שונות (C#, JavaScript, Python וכו') לקובץ טקסט יחיד
- **ניהול פלט** - בחירת שם קובץ הפלט וקטגוריה קבצים
- **עיצוב הפלט** - הוספת הערות מקור, סדור קבצים, הסרת שורות ריקות
- **אוטומציה** - יצירת קבצי תגובה (.rsp) לביצוע חוזר של פקודות

## דרישות

- **.NET 8.0** ומעלה
- `System.CommandLine` (מתוקן כבר בפרויקט)

## התקנה

1. שנשכפל או הורד את המיקום:
```bash
git clone <repository-url>
cd cli1
```

2. בנה את הפרויקט:
```bash
dotnet build
```

3. הרץ את הכלי:
```bash
dotnet run -- [commands]
```

## שימוש

### פקודה 1: `bundle`

אורז קבצי קוד לקובץ אחד.

#### סינטקס:
```bash
dotnet run -- bundle --language <languages> [options]
```

#### אפשרויות:

| אפשרות | שם קצר | תיאור | חובה |
|--------|--------|-------|------|
| `--language` | `-l` | רשימת שפות תכנות (cs, js, py, או all) | ✅ כן |
| `--output` | `-o` | שם קובץ ה-bundle המיוצא (ברירת מחדל: bundle.txt) | ❌ לא |
| `--note` | `-n` | הוספת הערות מקור לכל קובץ (ראה איפה הגיע) | ❌ לא |
| `--sort` | `-s` | סדר המיון: `name` (ברירת מחדל) או `type` | ❌ לא |
| `--remove-empty-lines` | `-r` | הסרת שורות ריקות מהקבצים | ❌ לא |
| `--author` | `-a` | שם היוצר (יופיע כהערה בראש הקובץ) | ❌ לא |

#### דוגמאות:

**דוגמה 1**: אריזת כל קבצי C#:
```bash
dotnet run -- bundle --language cs --output myCode.txt
```

**דוגמה 2**: אריזת כל הקבצים עם הערות מקור:
```bash
dotnet run -- bundle --language all --note --author "ישראל ישראלי"
```

**דוגמה 3**: אריזה עם סדור לפי סוג קובץ:
```bash
dotnet run -- bundle -l js,py -o output.txt --sort type --remove-empty-lines
```

**דוגמה 4**: אריזה קצרה (קיצור):
```bash
dotnet run -- bundle -l cs -o result.txt -n -r
```

### פקודה 2: `create-rsp`

יוצר קובץ תגובה (.rsp) עם פקודת bundle שנוצרה באופן אינטראקטיבי.

#### סינטקס:
```bash
dotnet run -- create-rsp
```

#### תיאור:
הפקודה מנחה אותך דרך סדרת שאלות יחידה:
1. שם קובץ התגובה (ללא סיומת)
2. שפות לכללות (למשל: cs,js או all)
3. שם קובץ הפלט
4. הוספת הערות מקור (y/n)
5. סדר מיון (name/type)
6. הסרת שורות ריקות (y/n)
7. שם היוצר (אופציונלי)

#### דוגמה:
```bash
dotnet run -- create-rsp
```

לאחר יצירת קובץ התגובה (למשל `mybundle.rsp`), תוכל להריץ את הפקודה כך:
```bash
dotnet run -- @mybundle.rsp
```

## הוצאי קבצים מדוללים

הכלי מדלל (מחריג) קבצים מהתיקיות הבאות באופן אוטומטי:
- `bin/`
- `obj/`
- `debug/`
- `release/`
- `node_modules/`

זה מבטיח שלא יתקבלו קבצי build או dependencies בלבלבוט.

## דוגמה שלמה של זרימת עבודה

### שלב 1: בדוק קבצי C# בתיקייה:
```bash
cd c:\path\to\project
dotnet run --project ../../cli1 -- bundle -l cs --note
```

### שלב 2: צור קובץ תגובה:
```bash
dotnet run --project ../../cli1 -- create-rsp
```

### שלב 3: השתמש בקובץ התגובה לאוטומציה:
```bash
dotnet run --project ../../cli1 -- @myconfig.rsp
```

## שגיאות נפוצות

### ❌ "אין קבצים תואמים"
**סיבה**: שום קבצים בשפה שנבחרה לא נמצאו בתיקייה.
**פתרון**: 
- וודא שמדובר בשפות הנכונות (cs, js, py וכו')
- בדוק שקבצים בעלי הסיומת הנכונה קיימים בתיקייה

### ❌ "ערך לא תקין לאפשרות --sort"
**סיבה**: הגדרת ערך שלא תקין ל-sort (חייב להיות `name` או `type`).
**פתרון**: השתמש בערכים הנכונים בלבד.

### ❌ "לא זכוי לחובה להזין שפה"
**סיבה**: הערך של `--language` ריק או חסר.
**פתרון**: ציין לפחות שפה אחת או "all".
